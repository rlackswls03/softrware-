# 프로젝트 업데이트: Adversarial Firewall 확장

## 1. 왜 방향을 바꿨는가

기존 프로젝트는 MNIST 분류 모델 4개에 대해 FGSM, FGSM adversarial training, 전이 공격, PGD 평가를 수행했다. 핵심 결과는 FGSM adversarial training이 같은 FGSM 공격에는 매우 강하지만 PGD와 전이 공격에서는 안정적으로 일반화되지 않는다는 점이었다.

GPT와의 대화 맥락에서는 물리적 적대적 패치나 직접 촬영 이미지 실험이 교수님에게 너무 뻔하게 보일 수 있다는 점이 문제였다. 그래서 새 공격이나 시연을 붙이는 대신, 기존 결과를 바탕으로 **공격받은 입력을 탐지하고, 정화하고, 위험하면 거부하는 시스템 수준 방어**로 발전시키는 방향을 선택했다.

최종 확장 제목은 다음과 같이 잡았다.

> Adversarial Firewall: 적대적 입력 탐지·정화·거부 기반 군 AI 영상인식 방어 파이프라인

## 2. 이번에 구현한 핵심 기능

이번 확장은 기존 `scripts.run_pipeline`을 깨지 않고, 별도 firewall pipeline으로 추가했다.

구현된 핵심 기능은 다음 4개다.

| 기능 | 구현 상태 | 설명 |
|---|---:|---|
| MNIST convolutional autoencoder | 완료 | clean MNIST 이미지를 복원하는 purifier/reformer |
| Reconstruction error detector | 완료 | `MSE(x, AE(x))`가 threshold보다 크면 적대적 입력 의심 |
| Purify and reclassify | 완료 | 공격 입력을 autoencoder로 정화한 뒤 classifier로 재분류 |
| Reject option policy | 완료 | 위험 점수가 높고 정화 후 confidence가 낮으면 `REJECT_SUSPICIOUS` 처리 |

추가로 연구 결과처럼 보이게 만드는 시각화와 요약 기능도 구현했다.

| 기능 | 구현 상태 | 설명 |
|---|---:|---|
| Detection ROC/AUC | 완료 | reconstruction error 기반 clean vs attack 탐지 ROC |
| TPR@FPR 5% | 완료 | clean 오탐률 5% 기준 탐지율 |
| score distribution plot | 완료 | clean/FGSM/PGD reconstruction error 분포 |
| accuracy recovery plot | 완료 | original accuracy, purified accuracy, final safe accuracy 비교 |
| decision breakdown plot | 완료 | 원본 통과/정화 후 통과/거부 비율 |
| image grid | 완료 | original / attacked input / purified 이미지 비교 |
| summary.md 자동 확장 | 완료 | `results/summary.md`에 Firewall 섹션 자동 추가 |

## 3. 새로 추가한 주요 파일

### 소스 코드

- `src/adversarial_mnist/autoencoder.py`
  - `ConvAutoencoder`
  - autoencoder checkpoint 저장/로드 helper
  - reconstruction loss helper

- `src/adversarial_mnist/firewall.py`
  - `reconstruction_error`
  - `choose_threshold_from_clean_validation`
  - `firewall_decision`
  - `ACCEPT_ORIGINAL`
  - `ACCEPT_PURIFIED`
  - `REJECT_SUSPICIOUS`
  - ROC/AUC helper

- `src/adversarial_mnist/firewall_evaluation.py`
  - threshold calibration
  - Clean/FGSM/PGD firewall evaluation
  - 샘플별 detection score 생성
  - 조건별 firewall result 생성
  - detection AUC summary 생성

### 실행 스크립트

- `scripts/train_autoencoder.py`
  - clean MNIST train set으로 autoencoder 학습
  - checkpoint 저장: `checkpoints/autoencoder_seed{seed}.pt`
  - history 저장: `results/raw/autoencoder_history.csv`

- `scripts/evaluate_firewall.py`
  - `smallcnn_standard`, `smallcnn_fgsm_at` 대상 firewall 평가
  - Clean, FGSM eps=0.25, PGD eps=0.25 조건 평가

- `scripts/plot_firewall_results.py`
  - firewall CSV를 읽어 그래프 생성
  - `results/summary.md`의 Firewall 섹션 업데이트

- `scripts/run_firewall_pipeline.py`
  - autoencoder 학습
  - threshold calibration
  - firewall 평가
  - 그래프 생성
  - summary 업데이트를 한 번에 실행

### 테스트

- `tests/test_autoencoder.py`
  - autoencoder output shape
  - output range `[0, 1]`
  - reconstruction loss scalar 확인

- `tests/test_firewall.py`
  - reconstruction error shape/value
  - threshold selection
  - reject policy decision
  - batch decision
  - AUC와 TPR@FPR 계산

## 5. 새 실행 명령

가장 중요한 명령은 이것이다.

```bash
python -m scripts.run_firewall_pipeline --quick --force
```

중간 규모 검증은 다음처럼 실행할 수 있다.

```bash
python -m scripts.run_firewall_pipeline --force --seeds 42 --test-subset 2000 --pgd-steps 3
```

개별 단계 실행도 가능하다.

```bash
python -m scripts.train_autoencoder --config configs/default.json --seeds 42 --force
python -m scripts.evaluate_firewall --config configs/default.json --seeds 42 --force
python -m scripts.plot_firewall_results --config configs/default.json
```

## 11. 보고서에 넣기 좋은 해석 문장

기존 실험에서 FGSM adversarial training은 FGSM 공격에는 높은 robust accuracy를 보였지만, PGD와 전이 공격에서는 일반화 한계를 보였다. 이에 본 확장에서는 모델 자체를 더 강건하게 만드는 접근을 넘어서, 입력 단계에서 적대적 입력을 탐지하고 autoencoder로 정화한 뒤, 위험도가 높으면 판단을 거부하는 Adversarial Firewall을 구현했다.

이 확장은 새로운 방어 알고리즘을 제안한다기보다, 공격 평가 프로젝트를 시스템 수준 방어 파이프라인으로 발전시킨 것이다. 특히 군 AI 적용 맥락에서는 모든 입력을 무조건 자동 분류하는 것보다, 위험 입력을 탐지하고 정화하거나 인간 검토로 넘기는 reject option이 실용적인 방어 절차로 해석될 수 있다.

단, 이 prototype은 adaptive attack에 대한 보장을 제공하지 않는다. 공격자가 autoencoder와 detector 구조를 알고 최적화할 경우 reconstruction error 기반 탐지는 우회될 수 있다. 따라서 본 결과는 완전한 보안 해법이 아니라, 입력 단계 탐지·정화·거부 절차의 가능성을 확인하는 MVP로 해석해야 한다.
